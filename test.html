jQuery Stuck Plugin demo

This jQuery plugin aims to facilitate keeping a sidebar item visible when the user scrolls past it, with the (entirely optional!) ability to constrain its movement within a container box so that it doesn’t interfere with a footer-type div. It also fires a series of custom events, allowing the developer to assign custom functions to the events (for the time being, it only fires one event. More are in the works.) Finally, it will also allow multiple objects to be stuck, so that section-specific sidebar items can stay with the user only as long as the relevant section is in view.

This plugin is still in its early stages, and is not recommended for production environments.

You’ll notice that this sidebar has CSS transitions applied to it (namely, background & box-shadow) that are fired when the fixed state is in effect. The target properties are specified in the .floating class, and are visible in Chrome, Safari, Firefox and Opera. IE should just show the change in styles, without the smooth transition. The point being that by applying classes we can make the sidebar appear different when it is fixed.

As implied above, various classes can be applied for both the fixed and bottom states. These classes stack on top of the default styles that are already applied by default in your stylesheet. In this demo, the .fixed class makes the element stick, while the .floating class specifies all other style properties. Useful for using the .fixed class throughout your document without being tied to the particular styles pertinent only to the sidebar.
